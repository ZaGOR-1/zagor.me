# Виправлення помилок i18n та статичної генерації

## Проблеми

### 1. Помилка: Invalid locale: service-worker.js
**Опис:** Браузер намагався завантажити `/service-worker.js`, але Next.js обробляв це як динамічний параметр `[locale]`, що призводило до помилки валідації локалі.

**Причина:** Файл service-worker.js не існував у папці `public`, тому запит потрапляв в обробку динамічного роуту `app/[locale]`.

### 2. Помилка: Missing param "/[locale]" in generateStaticParams()
**Опис:** При статичному експорті (`output: 'export'`) Next.js вимагає функцію `generateStaticParams()` для всіх динамічних роутів, включаючи кореневий `app/page.tsx`.

## Рішення

### 1. Створено файл service-worker.js
- Додано `public/service-worker.js` з базовою імплементацією
- Файл автоматично копіюється в `/out/` при збірці
- Тепер запити до `/service-worker.js` обробляються як статичний ресурс, а не як параметр локалі

### 2. Додано generateStaticParams() до кореневого роуту
- Додано експорт `generateStaticParams()` в `app/page.tsx`
- Функція повертає порожній масив, оскільки кореневий роут не має параметрів
- Це задовольняє вимоги Next.js для статичного експорту

## Результат
✅ Збірка проходить успішно без помилок
✅ Генеруються статичні сторінки для обох локалей (uk, en)
✅ Service worker доступний за адресою `/service-worker.js`
✅ Dev сервер запускається без помилок
